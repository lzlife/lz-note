# LZ Note

LZ Note 是一个运行在 ZTools 内的本地 Markdown 笔记插件，核心目标是“本地优先、Git 可同步、操作可预期”。

## 核心能力

- Markdown 编辑：基于 Vditor，支持所见即所得编辑。
- 文件树管理：支持新建、重命名、拖拽移动、删除与导入 Markdown。
- 名称检索：通过 ZTools `setSubInput` 对文件名/文件夹名进行实时过滤。
- 自动保存：在失焦、切换文件等时机自动落盘，降低内容丢失风险。
- Git 同步：支持拉取、推送与缺失文件确认策略。

## 同步策略说明

- 远程缺少本地已跟踪文件：可选择保留本地，或按远程删除。
- 本地缺少远程已跟踪文件：可选择从远程恢复到本地，或按本地删除。
- 修改冲突：当前策略为本地优先覆盖远程（与当前产品行为一致）。

## 使用说明

- 新建文件：先创建默认文件，再在编辑区顶部进入可重命名状态。
- 新建文件夹：保持文件树内创建与重命名流程。
- 删除确认：确认弹窗不支持点击蒙层关闭，避免误操作。

## 数据与格式

- 仅支持编辑 `.md` 文件。
- 非 Markdown 文件可在文件树中显示，但不可直接进入编辑。

## 常见问题

### 1) 为什么有时提示“本次没有发现需要同步的改动”？

说明本地与远程在当前分支上已经一致，或没有可提交的变化。

### 2) 为什么会出现同步前确认弹窗？

检测到本地与远程存在“缺失文件”差异时，会先让你选择策略，避免误删或误覆盖。

### 3) 删除后是否支持应用内恢复？

当前不提供独立恢复机制，建议通过 Git 历史回退。
